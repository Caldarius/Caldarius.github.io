面向对象(OOP)

面向过程（POP）& 面向对象(OOP)

  • 面向过程，强调的是功能行为，以函数为最小单位，考虑怎么做。

  • 面向对象，将功能封装进对 象，强调具备了功能的对象，以类/对象为最小单位，考虑谁来做。

  • 面向对象更贴近人类在日常的思维逻辑中采用的思想方法与原则，如 抽象、分类、继承、聚合、多态等。

OOP思想概述

三（四）大特征

  • 抽象（Abstract）

  • 封装 (Encapsulation)

  • 继承 (Inheritance)

  • 多态 (Polymorphism)

概述

  • 根据问题需要，选择问题所对应的实体。

    • 这个实体通常是现实世界有的，建议解决问题时向现实世界靠拢。同时，也要发挥自己的能动作用，合理构建理想事物

  • 从实体中寻找解决问题相关的属性和功能，这些属性和功能就形成了概念世界中的类。

  • 把抽象的实体用计算机语言进行描述，形成计算机世界中类的定义。即借助某种程序 语言，把类构造成计算机能够识别和处理的数据结构和算法。分别对应类的属性和行为。

  • 将类实例化成计算机世界中的对象。对象是我们解决问题的最终工具。

    Java 的 类(Class) 与 对象(Object)

二者的关系

  • 图纸和产品的关系。
  • 抽象和实体的关系。对象也成为类的实例，创建对象的过程也叫做类的实例化。
  • 类的设计是面向对象程序设计的重点。

类的构成

属性

对应类中的成员变量，用于描述某一类实体的属性。是构成程序的数据结构。

行为

对应类中的成员方法，用于描述一类实体的行为。是构成程序的算法。

数据结构+算法=程序

语法格式

public/protected/private class class_name{
    //属性；
    //方法；
}

// 例如：
public class Student{
    // 属性
    String Sno;
    String name;
    
    //方法
    public void hello(){
        System.out.println("Hello miao~");
    }
}
// class student 会作为后续的例子

对象的创建和使用

class_name object_name = new class_name(); //类名 对象名 = new 类名();
object_name.object_member; // 使用  对象名.对象成员  的方式访问对象成员（属性和方法）

public class RunTime{
    public static void main(String args[]){
        Student student = new Student();
        student.Sno = "201903010121";
        student.hello();
    }
}
  • 如果创建了一个类的 多个对象，对于类中 定义的属性，每个对 象都拥有各自的一套 副本，且互不干扰。后续的内存部分会介绍原理。

类的访问机制
  • 在一个类中的访问机制：类中的方法可以直接访问类中的成员变量。
  • 在不同类中的访问机制：先创建要访问类的对象，再用对象访问类中 定义的成员。
