计算机与二进制

在研究中……~

简述

  • 采用了0、1两个基本数字符号。
  • 进位规律——“逢二进一”
  • 位权——“2的整数次幂”

优点

  • 运算简单、物理实现容易、存储和传送方便可靠。
  • 在数字系统中普遍存在。

缺陷

  • 数字位数太长，字符单调，书写记忆以和阅读不方便。所以常常采用八进制或者十六进制来作为二进制数的缩写。

数值转换

多项式替代法

  • 将二进制表示数按位权展开，并运用十进制运算法则计算，所得结果即为对应十进制数。可推广到n进制转m进制。

基数乘除法

  • 整数转换——“除二取余”。

  • 小数转换——“乘二取整”

    • 例如：

      0.625 * 2 = 1.250

      0.250*2 = 0.500

      0.500*2 = 1.000

      将所得个位自上而下写出即可

      0.625 = 0.101（二进制）

二进制与八进制

2->8

以小数点为界，分别往高、低每三位一组，不足三位补0；

8->2

将美味八进制用三位二进制数表示，小数点位置保持不变。

原码

  • 符号位用0表示正，1表示负；数值位保持不变。

小数原码

原个位用作符号位。

例如：

​ X1 = 0.10，原码：0.10

​ X2 = -0.10，原码：1.10

  • 符号位的0/1代表着“负号”的有无。

    整数原码

在首位之前增加一位符号位

例如：

​ X1 = 101，原码：0101

​ X2 = -101，原码：1101

优点

  • 简单易懂，求取方便。

缺点

  • 加减运算不方便。
  • 当两数加减运算时，要根据运算以及参加运算的两个数的符号来确定是加还是减；
  • 如果做减法，首先要根据两数的大小来确定减数和被减数，以及运算结果的符号。
  • 综上所述，运算较为复杂。

反码

  • 符号位：用0表示正，1表示负；
  • 数值位：
    • 整数的反码与原码数值位相同；
    • 负数的反码是原码数值位按位取反。

优点

  • 采用反码进行加减运算时，无论进行两数相加还是两数相减，均可通过加法实现。

  • 运算时，符号位和数值位一样参加运算。当符号位产生进位时，应该将进位加到运算结果的最低位，得到运算结果。

  • [X1+X2]反 = [X1]反 + [X2]反

  • [X1-X2]反 = [X1]反 + [-X2]反 (运算时将符号位取反即可)

    例如：

    ​ X1 = +0.1110 X2 = +0.0101

    ​ X1-X2 = 0.1110+1.1010 = 0.1001

补码

  • 符号位：用0表示正，1表示负；
  • 数值位：
    • 整数的反码与原码数值位相同；
    • 负数的反码是原码数值位按位取反，末位+1。

优点

  • 补码优势与反码相同。

  • 运算时，符号位和数值位一样参加运算。当符号位产生进位时，应去掉进位，得到运算结果。

  • 补码还原真值时，对数值位取反+1。

BCD码（Binary Coded Decimal）

用四位二进制代码对十进制数字符号进行编码——二—十进制代码。

BCD码并不是二进制数，而是采用二进制编码的十进制数。

8421

  • 是用4位二进制码表示一位十进制字符的一种有权码，4位二进制码从高位至低位的权依次为23、22、21、20，即为8、4、2、1,故称为8421码
  • 按8421码编码的0～9与用4位二进制数表示的0～9完全一样。所以，8421码是一种人机联系时广泛使用的中间形式。

注意

  • 8421码中不允许出现1010～1111六种组合(因为没有十进制数字符号与其对应)。
  • 十进制数字符号的8421码与相应ASCII码的低四位相同，这一特点有利于简化输入输出过程中BCD码与字符代码的转换。

2421

  • 2421码: 是用4位二进制码表示一位十进制字符的一种有权码，4位二进制码从高位至低位的权依次为2、4、2、1,故称为2421码。

  • 若一个十进制字符X的2421码为a3 a2 a1 a0，则该字符的值为：

    X = 2a3 + 4a2 + 2a1 + 1a0

  • 2421码与十进制数之间的转换同样是按位进行的，例如，
    (258)10 = (0010 1011 1110)2421码
    (0010 0001 1110 1011)2421码 = (2185)10

注意

  • 2421码不具备单值性。例如，0101和1011都对应十进制数字5。为了与十进制字符一一对应，2421码不允许出现0101～1010的6种状态。

  • 2421码是一种对9的自补代码。即一个数的2421码只要自身按位变反，便可得到该数对9的补数的2421码。

    0100->1011 (4->5)

余三

  • 是由8421码加上0011形成的一种无权码，由于它的每个字符编码比相应8421码多3，故称为余3码。

注意

  • 余3码中不允许出现0000、0001、0010、1101、1110和1111六种状态。
  • 余3码与十进制数进行转换时，每位十进制数字的编码都应余3。（转换之后或者之前先 "-3"）
  • 余3码也是一种对9的自补代码。
  • 两个余3码表示的十进制数字相加时，能产生正确进位信号，但对“和”必须修正。
    • 修正的方法是：如果有进位，则结果加3；如果无进位，则结果减3。
